# Coze Workflow Canvas JSON Format (DSL Specification)

Extracted from Coze Studio source code.

Source: `backend/domain/workflow/entity/vo/canvas.go`, `backend/domain/workflow/entity/node_meta.go`,
`frontend/packages/workflow/sdk/__tests__/__mock_data__/canvas-schema.ts`

## Top-Level Structure

```json
{
  "nodes": [ ... ],
  "edges": [ ... ],
  "versions": { "loop": "v2" }
}
```

**Go type** (`vo.Canvas`):

| Field      | Type      | JSON key     | Description                          |
|------------|-----------|--------------|--------------------------------------|
| Nodes      | `[]*Node` | `nodes`      | All nodes in the workflow            |
| Edges      | `[]*Edge` | `edges`      | Top-level edges connecting nodes     |
| Versions   | `any`     | `versions`   | Schema version metadata              |

---

## Node Object

**Go type** (`vo.Node`):

```json
{
  "id": "100001",
  "type": "1",
  "meta": {
    "position": { "x": 0, "y": 0 }
  },
  "data": { ... },
  "blocks": [ ... ],
  "edges": [ ... ],
  "version": "3"
}
```

| Field   | Type      | JSON key           | Description                                                                    |
|---------|-----------|--------------------|--------------------------------------------------------------------------------|
| ID      | `string`  | `id`               | Unique node ID within the workflow, generated by frontend. Entry=`100001`, Exit=`900001`. |
| Type    | `string`  | `type`             | **Numeric string** corresponding to the NodeTypeMeta `ID` field (see table below). |
| Meta    | `any`     | `meta`             | Frontend metadata, primarily `position: {x, y}`.                               |
| Data    | `*Data`   | `data`             | Node configuration: inputs, outputs, nodeMeta.                                 |
| Blocks  | `[]*Node` | `blocks,omitempty` | Sub-nodes for composite nodes (Loop, Batch).                                   |
| Edges   | `[]*Edge` | `edges,omitempty`  | Edges within composite nodes (Loop, Batch).                                    |
| Version | `string`  | `version,omitempty`| Schema version of this node type.                                              |

### Node `meta`

```json
{
  "position": { "x": 0, "y": 0 }
}
```

Only `position` is observed in practice. Coordinates use a 2D canvas system (see edge-and-layout.md).

### Node `data` (Data struct)

```json
{
  "nodeMeta": {
    "title": "Start",
    "description": "...",
    "icon": "icon-Start-v2.jpg",
    "subTitle": "",
    "mainColor": "#5C62FF"
  },
  "outputs": [ ... ],
  "inputs": { ... },
  "size": { "width": 360, "height": 200 }
}
```

| Field    | JSON key           | Type            | Description                                                |
|----------|--------------------|-----------------|------------------------------------------------------------|
| Meta     | `nodeMeta`         | `NodeMetaFE`    | Display metadata: title, description, icon, subTitle, mainColor. |
| Outputs  | `outputs`          | `[]any`         | Output fields. Usually `[]*Variable`; composite nodes use `[]*Param`. |
| Inputs   | `inputs`           | `*Inputs`       | All input configuration for the node.                      |
| Size     | `size`             | `any`           | Node size override. Only used by Comment nodes.            |

## Inputs Object

**Go type** (`vo.Inputs`):

```json
{
  "inputParameters": [ ... ],
  "settingOnError": { ... },
  "chatHistorySetting": { ... },
  "batch": { ... },
  "llmParam": [ ... ],
  "terminatePlan": "returnVariables",
  "code": "...",
  "language": 5,
  ...
}
```

| Field               | JSON key              | Description                                              |
|---------------------|-----------------------|----------------------------------------------------------|
| InputParameters     | `inputParameters`     | User-defined input fields as `[]*Param`.                 |
| ChatHistorySetting  | `chatHistorySetting`  | Chat history config (chatflow mode).                     |
| SettingOnError      | `settingOnError`      | Error handling: processType, retryTimes, timeoutMs.      |
| NodeBatchInfo       | `batch`               | Batch mode config (per-node batch, not NodeTypeBatch).   |
| LLMParam            | `llmParam`            | LLM configuration params. Either `[]*Param` or `map`.   |

Plus embedded type-specific fields (e.g. `code`, `language` for CodeRunner; `terminatePlan` for Exit; `branches` for Selector; etc.).

## Variable (Output Field)

**Go type** (`vo.Variable`):

```json
{
  "name": "output",
  "type": "string",
  "required": false,
  "assistType": 0,
  "schema": null,
  "description": "",
  "defaultValue": ""
}
```

| Field        | JSON key       | Type           | Description                                               |
|--------------|----------------|----------------|-----------------------------------------------------------|
| Name         | `name`         | `string`       | Field name.                                               |
| Type         | `type`         | `VariableType` | Data type (see enum below).                               |
| Required     | `required`     | `bool`         | Whether the field is required.                            |
| AssistType   | `assistType`   | `int64`        | Secondary type for string fields (file subtypes, time).   |
| Schema       | `schema`       | `any`          | For `object`: `[]*Variable`; for `list`: `*Variable`.     |
| Description  | `description`  | `string`       | Description (used on Entry node for workflow tools).      |
| DefaultValue | `defaultValue` | `any`          | Default value (effective only on Entry node).             |

### VariableType Enum

| Value       | Constant               |
|-------------|------------------------|
| `"string"`  | `VariableTypeString`   |
| `"integer"` | `VariableTypeInteger`  |
| `"float"`   | `VariableTypeFloat`    |
| `"boolean"` | `VariableTypeBoolean`  |
| `"object"`  | `VariableTypeObject`   |
| `"list"`    | `VariableTypeList`     |

### AssistType Enum (for string fields)

| Value  | Meaning  |
|--------|----------|
| 0      | Not set  |
| 1      | Default  |
| 2      | Image    |
| 3      | Doc      |
| 4      | Code     |
| 5      | PPT      |
| 6      | TXT      |
| 7      | Excel    |
| 8      | Audio    |
| 9      | Zip      |
| 10     | Video    |
| 11     | Svg      |
| 12     | Voice    |
| 10000  | Time     |

## Param (Input Field with Source)

**Go type** (`vo.Param`):

```json
{
  "name": "input",
  "input": {
    "type": "string",
    "value": {
      "type": "ref",
      "content": {
        "source": "block-output",
        "blockID": "100001",
        "name": "input"
      },
      "rawMeta": { "type": 1 }
    }
  }
}
```

| Field     | JSON key    | Description                                            |
|-----------|-------------|--------------------------------------------------------|
| Name      | `name`      | Field name.                                            |
| Input     | `input`     | BlockInput for normal singular fields.                 |
| Left      | `left`      | Left side of an expression (e.g. variable assignment). |
| Right     | `right`     | Right side of an expression.                           |
| Variables | `variables` | Group of fields (NodeTypeVariableAggregator only).     |

---

## BlockInput (Value Source)

**Go type** (`vo.BlockInput`):

```json
{
  "type": "string",
  "value": {
    "type": "literal",
    "content": "hello world"
  }
}
```

| Field      | JSON key     | Type                 | Description              |
|------------|--------------|----------------------|--------------------------|
| Type       | `type`       | `VariableType`       | Data type of the value.  |
| AssistType | `assistType` | `int64`              | Secondary string type.   |
| Schema     | `schema`     | `any`                | Sub-schema for complex types. |
| Value      | `value`      | `*BlockInputValue`   | The actual value.        |

### BlockInputValue

| Field   | JSON key  | Description                                               |
|---------|-----------|-----------------------------------------------------------|
| Type    | `type`    | One of: `"literal"`, `"ref"`, `"object_ref"`.             |
| Content | `content` | For `literal`: raw value (string/number/bool). For `ref`: `BlockInputReference`. |
| RawMeta | `rawMeta` | Frontend metadata (e.g. `{"type": 1}`).                   |

### BlockInputReference (for `ref` type)

```json
{
  "source": "block-output",
  "blockID": "100001",
  "name": "input",
  "path": []
}
```

| Field   | JSON key  | Description                                          |
|---------|-----------|------------------------------------------------------|
| BlockID | `blockID` | ID of the source node.                               |
| Name    | `name`    | Output field name on the source node.                |
| Path    | `path`    | Path for nested field access (array of strings).     |
| Source  | `source`  | Reference source type (see below).                   |

### RefSourceType Enum

| Value                    | Description                                      |
|--------------------------|--------------------------------------------------|
| `"block-output"`         | References output of another node.               |
| `"global_variable_app"`  | References an app-level global variable.          |
| `"global_variable_system"` | References a system global variable.           |
| `"global_variable_user"` | References a user-scoped global variable.        |

---

## Edge Object

**Go type** (`vo.Edge`):

```json
{
  "sourceNodeID": "100001",
  "targetNodeID": "172934",
  "sourcePortID": "",
  "targetPortID": ""
}
```

| Field        | JSON key         | Description                                           |
|--------------|------------------|-------------------------------------------------------|
| SourceNodeID | `sourceNodeID`   | ID of the source node.                                |
| TargetNodeID | `targetNodeID`   | ID of the target node.                                |
| SourcePortID | `sourcePortID`   | Output port identifier (empty string = default port). |
| TargetPortID | `targetPortID`   | Input port identifier (empty string = default port).  |

See `edge-and-layout.md` for port naming conventions.

### Composite Node Types

These node types contain sub-nodes in `blocks` and sub-edges in `edges`:
- **Loop** (ID=21) -- `IsComposite: true`
- **Batch** (ID=28) -- `IsComposite: true`

## Error Handling (SettingOnError)

```json
{
  "switch": true,
  "processType": 1,
  "retryTimes": 0,
  "timeoutMs": 60000,
  "dataOnErr": "{\"output\": \"\"}"
}
```

| ProcessType | Constant                       | Behavior                              |
|-------------|--------------------------------|---------------------------------------|
| 1           | `ErrorProcessTypeThrow`        | Throws the error as usual.            |
| 2           | `ErrorProcessTypeReturnDefaultData` | Returns `dataOnErr` value.       |
| 3           | `ErrorProcessTypeExceptionBranch`   | Executes the exception branch.   |

## Complete Minimal Example

A workflow with Start -> LLM -> End:

```json
{
  "nodes": [
    {
      "id": "100001",
      "type": "1",
      "meta": {
        "position": { "x": 0, "y": 0 }
      },
      "data": {
        "nodeMeta": {
          "title": "Start",
          "description": "The starting node of the workflow.",
          "icon": "icon-Start-v2.jpg",
          "subTitle": ""
        },
        "outputs": [
          {
            "type": "string",
            "name": "input",
            "required": false
          }
        ]
      }
    },
    {
      "id": "200001",
      "type": "3",
      "meta": {
        "position": { "x": 500, "y": 0 }
      },
      "data": {
        "nodeMeta": {
          "title": "LLM",
          "description": "Invoke the large language model.",
          "icon": "icon-LLM-v2.jpg",
          "mainColor": "#5C62FF",
          "subTitle": "LLM"
        },
        "inputs": {
          "inputParameters": [
            {
              "name": "input",

                    "source": "block-output",

                  }

            }

    "loop": "v2"
  }
```

## Chatflow Mode Differences

In chatflow mode, the Entry node outputs `USER_INPUT` (required) and `CONVERSATION_NAME`, and the Exit node uses `terminatePlan: "useAnswerContent"` with `streamingOutput: true` instead of `returnVariables`.
